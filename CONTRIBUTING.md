# 기여 가이드

모든 작업은 **이슈 생성 → 브랜치 → 커밋 → PR** 순서로 진행합니다.

## 커밋 컨벤션

[Conventional Commits](https://www.conventionalcommits.org/) 형식을 따릅니다.

```
<type>: <제목>

<본문(선택)>

<푸터(선택)>
```

- **type**: 영어 소문자 (아래 표)
- **제목**: 한국어, 명령형으로 간결하게. 끝에 마침표 X. 50자 이내 권장
- 본문: 무엇을·왜 바꿨는지 (어떻게는 코드로). 한 줄 띄우고 작성
- 푸터: 관련 이슈 연결 (`Closes #12`, `Refs #34`)

### type 목록

| type       | 설명                                           |
| ---------- | ---------------------------------------------- |
| `feat`     | 새로운 기능 추가                               |
| `fix`      | 버그 수정                                      |
| `docs`     | 문서 수정 (README, 주석 등)                    |
| `style`    | 코드 포맷팅, 세미콜론 등 동작에 영향 없는 변경 |
| `refactor` | 동작 변경 없는 코드 구조 개선                  |
| `test`     | 테스트 추가/수정                               |
| `chore`    | 빌드, 설정, 의존성 등 기타 잡일                |

### 예시

```
feat: 메모 CRUD API 추가
fix: 메모 저장 시 500 에러 수정
docs: README에 로컬 실행 방법 추가
refactor: MemoService 의존성 정리
chore: gitignore에 .DS_Store 추가
```

이슈와 연결하는 경우:

```
feat: 메모 CRUD API 추가

메모를 저장/조회/수정/삭제하는 REST API를 구현한다.

Closes #12
```

## 브랜치 전략

작업은 이슈 단위로 브랜치를 만들어 진행합니다.

```
<type>/#<이슈번호>-<간단한-설명>
```

예시:

```
feat/#12-memo-crud
fix/#23-save-500-error
docs/#5-readme
```

- `main`에 직접 push 하지 않고, PR을 통해 머지합니다.

## PR 컨벤션

- **제목**: 커밋과 동일하게 Conventional Commits 형식 (예: `feat: 메모 CRUD API 추가`)
- PR 생성 시 자동으로 채워지는 템플릿을 작성합니다.
- 본문에 **관련 이슈를 반드시 연결**합니다 (`Closes #12`) — 머지 시 이슈가 자동으로 닫힙니다.
- 리뷰 승인 후 머지합니다.
